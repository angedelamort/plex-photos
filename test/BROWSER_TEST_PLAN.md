# plex-photos — Browser Test Plan

This plan validates the running app end-to-end through a real browser, in
**mock auth mode** (no Plex server needed). It complements the Go unit and
integration tests in `test/unit` and `test/integration`.

## Preconditions

- The dev server is started in mock mode on a known port (default `8099`):
  ```
  AUTH_PROVIDER=mock MOCK_USER=dev MOCK_ADMIN=true \
  DATA_PATH=./testdata/data PORT=8099 \
  go run .
  ```
- Seed images exist under `./testdata/photos` (generated by `go run testdata/gen/gen.go`).
- Base URL: `http://localhost:8099`.

## Test cases

Each step lists an Action and the Expected result. Capture a screenshot for any
failed step.

### TC1 — Login screen / authenticated bootstrap
- Action: Navigate to `http://localhost:8099/`.
- Expected: If not authenticated, the login screen shows the "Se connecter avec Plex" button.
  After clicking it (mock auto-logs in), the app shell appears with the topbar
  logo "plex-photos" and an avatar showing initials (e.g. "DE" for "dev").

### TC2 — Admin can create a library
- Action: Click the sidebar "Admin → Librairies", then "Nouvelle librairie".
  Fill Name = `Famille`, Dossier root = `./testdata/photos/famille`,
  Whitelist = `dev`. Save.
- Expected: A new library row "Famille" appears with its root path and whitelist.

### TC3 — Scan populates collections/albums
- Action: On the "Famille" row, click "Scanner".
- Expected: The pill updates to show "1 collection(s)" (montreal). No error alert.

### TC4 — Sidebar reflects libraries
- Action: Observe the sidebar "Mes librairies" section.
- Expected: "Famille" is listed. Clicking it shows the library view with a
  "montreal" collection card showing "2 album(s)".

### TC5 — Navigate Library → Collection → Album
- Action: Click "montreal", then the "carnaval-2025" album card.
- Expected: Breadcrumb shows Famille › montreal › carnaval-2025. A photo grid
  with 3 thumbnails renders (thumbnails load as images, not placeholder icons).

### TC6 — Viewer prev/next + keyboard
- Action: Click the first photo to open the full-screen viewer. Click the next
  arrow, then press the Left/Right arrow keys, then Escape.
- Expected: The viewer opens with the original image, caption shows "name (n/total)",
  navigation cycles through photos, and Escape closes the viewer.

### TC7 — Slideshow
- Action: In an album, click "Slideshow".
- Expected: The viewer opens and auto-advances every few seconds. Clicking the
  slideshow button again (or a nav arrow) stops auto-advance.

### TC8 — Admin sets a cover from the viewer
- Action: Open a photo in the viewer (as admin) and click "Définir comme couverture".
- Expected: A confirmation state appears ("Couverture définie"). Returning to the
  collection shows that photo as the album's cover thumbnail.

### TC9 — Logout
- Action: Click the logout icon in the topbar.
- Expected: Redirected to `/`, the login screen is shown again. Reloading does
  not restore the session.

### TC10 — Non-admin restrictions (optional, requires MOCK_ADMIN=false)
- Action: Restart the server with `MOCK_ADMIN=false`, reload, log in.
- Expected: The "Admin" sidebar section is hidden. The "Définir comme couverture"
  button does not appear in the viewer.

## Pass criteria

All TC1–TC9 pass (TC10 optional). Any console error in the browser devtools or
failed network request (non-2xx on `/api/*`) is a failure.
